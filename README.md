# Linguistic Event Matching for fNIRS Analysis

A Python pipeline for pairing stuttered speech events with linguistically matched fluent twins, enabling event-related fNIRS analysis that isolates the neural signature of stuttering by regressing out linguistic and motoric noise.

---

## Background

This tool implements the **Linguistic Event Matching** method. Given a transcript of naturalistic speech from a Person Who Stutters (PWS), Linguistic Event Matching pairs each stuttered word with its most linguistically similar fluent "twin" from the same recording. The resulting pairs are used as matched event contrasts in fNIRS GLM analysis.

The matching logic prioritizes:
1. **Temporal isolation** — fluent candidates must not fall within 5 seconds of any stuttered event
2. **Linguistic similarity** — scored across grammatical class, proper noun status, phoneme manner, and syllable count (0–4 points)
3. **Frequency proximity** — used as a tiebreaker when similarity scores are tied

## Input Format

Your transcript must be a CSV file with exactly these four columns (case-sensitive):

| Column | Type | Description |
|---|---|---|
| `word` | string | The spoken word |
| `start_time` | float | Word onset in seconds |
| `end_time` | float | Word offset in seconds |
| `label` | string | Must be exactly `Stuttered` or `Fluent` |

**Example:**
```
word,start_time,end_time,label
the,0.52,0.71,Fluent
cat,1.10,1.45,Stuttered
sat,1.50,1.80,Fluent
```

## Usage

### Step 1 — Generate the linguistic lookup table

```bash
python3 lem_lookup.py --transcript your_transcript.csv
```

This reads every unique word in your transcript and automatically generates `linguistic_lookup.csv` using:
- **spaCy** — grammatical class (Content vs. Function) and proper noun detection
- **CMU Pronouncing Dictionary** (via nltk) — syllable count and phoneme manner of articulation
- **wordfreq** — word frequency on the Zipf scale

Any words not found in the CMU dictionary (e.g. proper names, disfluencies) will be flagged in the terminal output with their syllable count defaulted to 1. These can be manually corrected in `linguistic_lookup.csv` before running Step 2.

### Step 2 — Run matching

```bash
python3 lem_main.py --transcript your_transcript.csv --lookup linguistic_lookup.csv
```

**Optional: adjust the buffer window (default is 5.0 seconds)**
```bash
python3 lem_main.py --transcript your_transcript.csv --lookup linguistic_lookup.csv --buffer 3.0
```

## Output

### `matched_pairs.csv`
Each row is a matched stutter–twin pair. Columns include:

| Column | Description |
|---|---|
| `stutter_word` / `twin_word` | The matched words |
| `stutter_start` / `stutter_end` | Stutter event timing (seconds) |
| `twin_start` / `twin_end` | Twin event timing (seconds) |
| `stutter_gram_class` / `twin_gram_class` | Grammatical class |
| `stutter_proper` / `twin_proper` | Proper noun status |
| `stutter_manner` / `twin_manner` | Phoneme manner of articulation |
| `stutter_syllables` / `twin_syllables` | Syllable count |
| `stutter_freq` / `twin_freq` | Word frequency (Zipf scale) |
| `similarity_score` | Match quality (0–4) |
| `distance_score` | Frequency distance (tiebreaker; lower = better) |

### `unmatched_log.csv`
Generated only if some stuttered events could not be matched. Lists the word, timing, and reason for failure.

## Matching Algorithm

1. **Buffer filter** — Any fluent word within 5 seconds of a stuttered event (on either side) is excluded from the candidate pool.
2. **Similarity scoring** — Each remaining fluent candidate is scored 0–4 against the stuttered word across: Grammatical Class, Proper Noun Status, Phoneme Manner, and Syllable Count (+1 per match).
3. **Tiebreaker** — If multiple candidates share the top score, the one with the smallest absolute difference in Word Frequency (Zipf scale) is selected.
4. **Greedy matching** — Stuttered events are matched in order of scarcity (those with the fewest perfect 4-point candidates are matched first). Once a fluent word is claimed, it cannot be reused.
